# Problem Description 題敘:

Link

ZJ 也有一樣的題目 (不過還是以矩陣這題來說明)

兩個矩陣 A 與 B 要相乘，必須滿足 A 的行數等於 B 的列數，若 A 是 p * q 的矩陣，B

是 q * r 的矩陣，則 A * B 的結果是一個 p*r 的矩陣，而需要的純量乘法數假設是 p * q * r 。

矩陣乘法滿足結合律，也就是說 A * B * C = (A * B) * C = A * (B * C)，可是不同順序所需要的純量乘法數不同。

因此請找出最好的相乘順序使得使用的純量乘法數的總和最少。

# Thinking 思考歷程:

嗯，沒有明確計算順序耶，好難想 ... 狀態也列不太出來，燒雞。只知道直接假設 dp[][] 為所需的乘法次數就沒了。沒辦法了，舉白旗，於是去看了一下題解，後來發現題目原來只要求 O(n3)O(n^3)O(n3) 的解就好，那是不是該枚舉甚麼呢？如果 dpdpdp 沒有名顯計算順序可以直接轉移過來的話，那應該可以跟分治一樣，讓我當前的區間是由兩個較小區間相乘後轉移過來的？似乎是可以這麼做？

# Solution 題解:

此題是區間 DP 經典題。個人有那種類似分治的 "感覺" (只不過在這裡沒有平均切成兩半啦！) ，先假設我們要算的是 dp[L][R] ， (這裡採左閉右閉，LLL 跟 RRR 各代表給定序列中其中一個數字的索引，例: [3][4] 及是一個矩陣)，接著可以枚舉一個 KKK ，始得:

dp[L][R]=min(dp[L][R], dp[L][K]+dp[K][R]+兩個較小矩陣相乘的 cost),  ∀ K∈L∼R−1dp[L][R] = min(dp[L][R],\ dp[L][K] + dp[K][R] + 兩個較小矩陣相乘的\ cost), \ \ \forall\ K\in L\sim R-1 dp[L][R]=min(dp[L][R], dp[L][K]+dp[K][R]+兩個較小矩陣相乘的 cost),  ∀ K∈L∼R−1

寫完了？但等等！對於那兩個矩陣的 costcostcost 怎麼辦？？該不會還是要一個一個乘起來吧？這樣複雜度挺爛的耶... 那應該不太需要這麼做。根據題目，我們不能改變矩陣序列的順序，只能改變相乘的順序，因此 costcostcost 其實就是:

矩陣L 的左端×矩陣K 的右端 (or 矩陣K+1 的左端)× 矩陣R 的右端矩陣_L\ 的左端 \times 矩陣_K\ 的右端\ (or\ 矩陣_{K+1}\ 的左端) \times\ 矩陣_R\ 的右端 矩陣L​ 的左端×矩陣K​ 的右端 (or 矩陣K+1​ 的左端)× 矩陣R​ 的右端

OK! 那接下來就是要處理的部分，就是找 Bottom upBottom\ upBottom up 的計算順序。不過對於這題我真心較喜歡 Top downTop\ downTop down ，因為在這題的它更直覺，且好寫多了 (底下也有 top down Code)。 那 ，Bottom upBottom\ upBottom up 順序又是怎麼做的呢？回顧一下，我們用了 dp[L][R]dp[L][R]dp[L][R] 來假設我們已經擁有了某個區間，接著才枚舉 KKK 的位置。那我們回過頭來想，這個 "區間" 又要怎麼訂出來呢？肯定也是枚舉 LLL 跟 RRR 啊！但且慢，仔細想一下，如果依照一般的二維陣列計算順序會吃到還沒計算過的值喔，那又該怎麼辦？想想剛剛的計算順序為何會出問題。
一個可行個想法是枚舉長度，也就是沒舉所有 222 個相鄰矩陣相乘的結果，接著枚舉所有 333 個相鄰矩陣相乘的結果，再枚舉所有 444 個相鄰矩陣相乘的結果.... 一直做到 nnn ，也枚舉所有 n−1n-1n−1 個相鄰矩陣相乘的結果。

一旦有了區間的長度，就能枚舉 LLL 跟RRR 了！如此就不會有提取沒算過值的情形囉！
