# Problem Description 題敘:

Link

Easy Version - Link (單調對列優化)

  1. 由下而上給你 nnn 隻烏龜，你可以挑選一些烏龜，把他們融合起來成一隻，如此，他們的違和度也會相加。
  2. xxx 個烏龜合出的大烏龜會有強度 xxx 的違和光芒。(斜率優化版才有)
  3. 一隻違和度 ccc 的烏龜如果放在烏龜塔的第 mmm 層 ，看起來會有 c×(m−1)c\times(m-1)c×(m−1) 的違和度。(層 = 融合後，由下而上的數第 mmm 隻烏龜)
  4. 烏龜塔的違和度是所有烏龜的違和度減掉所有違合光芒的強度。(斜率優化版才需考慮光芒)
  5. 為了避免烏龜塔太矮，每個要融合的連續區段不能包含超過 kkk 隻烏龜。
  • 簡易版與困難版唯一的不同就是 "光芒的有無"。

# Solution 題解:

這題真的比較難​，有滿多的門檻要跨過。光轉移就不是很好理解，題解也看了好久才有一點頭緒。我們先此題的簡單版下手，最後再把斜率優化加上去就是這題的正解。

# 狀態與轉移 💡​

這個轉移式可能不太好理解，我們慢慢來。

可以觀察一個很特別的條件。違和度是乘上第 m−1m-1m−1 層，這意味著，若是第 111 層的話，根本不用算，因為 c×(1−1)=0c \times (1-1) = 0c×(1−1)=0 。如果我們正的做 dpdpdp ，越往上，要乘的層數數字會越來越大... 這.... 好像有點難算。

那... 如果倒著做 (由上而下) 呢？或許會想說: 「阿 可是這樣不是只是先乘大的層數再乘小的層數，有差嗎？-_-」，其實是有差的喔，而且若倒著做，計算方式不必那麼複雜。

⟸i增加的方向，由後往前dp\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \Large\Longleftarrow i\ 增加的方向，由後往前\ dp \\          ⟸i 增加的方向，由後往前 dp

c1,c2,c3,c4\Large c_1,\ c_2,\ c_3,\ c_4c1​, c2​, c3​, c4​

觀察到這個條件後，我們就能定義狀態: dp[i]⟹dp[i] \Longrightarrowdp[i]⟹ 以 iii 為最底層，考慮前 0∼i0 \sim i0∼i 的最佳解。

之後就是轉移，我們可以得到:

dp[i]=maxdp[j]+Sum[j],for(i−k≤j≤i−1)\Large dp[i] = max\\{\ dp[j] + Sum[j]\ \\}, for\ (i-k \leq j \leq i -1)dp[i]=max dp[j]+Sum[j] ,for (i−k≤j≤i−1)

其中， jjj 的範圍的訂定是由於題目連續區段不能超過 KKK 的限制。而 倒著做 dpdpdp 的精髓是，我們當前枚舉到的 j∼ij \sim ij∼i 隻是 最底層 的，所以轉移中我們只需要枚舉他的數量，再加上之前算過的答案。

又由於我們是從上往下 (序列的後面做到前面) 做 dpdpdp ，最底層那堆的違和度會 ×0\times 0×0 ，因此，不管烏龜多大隻或多小隻，跟本不用去考慮他的違和度。

踩在上述的特性上，每前進一個 jjj ，其實就是 dp[j]dp[j]dp[j] 的答案 再多一個違和度前綴和 S[j]S[j]S[j]。為甚麼？

想像一下，因為多枚舉了最底層 (第 000 層)，那原本的最底層 (第 000 層) ⟹\Longrightarrow⟹ 第 111 層，第 222 層 ⟹\Longrightarrow⟹ 第 333 層 ... 第 nnn 層 ⟹\Longrightarrow⟹ 第 n+1n+1n+1 層。層數一起都往上挪動而增加了 111 ，且每個烏龜又最多只能被融合一次，因此不管之前的分堆有多麼的複雜，都不用去理他。有點像分配律的概念，反正要求的是烏龜塔違和度 總和，所以我們只要加上一個前綴和，剛剛好。 --- Code-1

# 單調隊列優化 💡​

講完了轉移，我們再回顧一下遞迴式，我們發現，maxmaxmax 裡面的東西只跟 jjj 有關係，且每次要求 極值 (最大值)，這個是一個單調隊列使用的觀察。我們看到在 iii 一直往左走的同時，jjj 會往右看，是一個有 時間性 的區間範圍。這是甚麼意思呢？在我們的區間內若有一個最大值 xxx，只要我們的 xxx 跟 iii 的距離超過了題目給定了 KKK ，他就超過了時限，不能再是最大值了。

於是我們用 deque 維護一個單調遞減的序列，我們也能看到，如果當前新的 jjj 值比當前 deque 最右端的 jjj 值還大的話，那麼最右端的 jjj 值就沒有用了，可以直接 pop_back() 。更進一步，若新的值實在太大，還可能一路往前砍掉許多值，直到 deque 最右端的元素比他還大，或者 deque 為空。最後再把新的 jjj 值推入 deque 。

也因為這個單調性的性質，我們每次取用最大值都能用 dq.front() 在 O(1)O(1)O(1) 的時間得到，且每個值只會進、出 deque 各一次，因此我們就能把複雜從 O(n2)O(n^2)O(n2) 優化到 O(n)O(n)O(n) 。 --- Code-2

# 斜率優化 💡​

重頭戲來了。剛剛單調對列的解法只能用來處理簡單版的烏龜疊疊樂，這題原本預設的解是斜率優化，比簡單版多了要扣掉光芒這個條件，因此，我們必須先改一下轉移式:

dp[i]=maxdp[j]+Sum[j]−(i−j)2,for(i−k≤j≤i−1)\large dp[i] = max\\{\ dp[j]\ + Sum[j]\ - (i-j)^2\ \\},\ for\ (i-k\leq j\leq i-1)dp[i]=max dp[j] +Sum[j] −(i−j)2 , for (i−k≤j≤i−1)

這樣就是我們的 O(n2)O(n^2)O(n2) 的算法，直接 submit 上去顯然會 TLE. 我們想想能不能用單調對列優化... 發現好像不行耶！maxmaxmax 裡面的東西不只有 jjj ，有 (i−j)2(i-j)^2(i−j)2 這個鬼東西，可惡，怎麼辦？

又作一些觀察，我們先展開差的平方公式 (i−j)2(i-j)^2(i−j)2 ，再把只跟 iii 相關的值拿出 maxmax\\{\\}max，可得到遞迴式:

dp[i]=maxdp[j]+Sum[j]+2ij−j2−i2,for(i−k≤j≤i−1)\large dp[i] = max\\{\ dp[j]\ + Sum[j]\ + 2ij - j^2\ \\} - i^2,\ for\ (i-k\leq j\leq i-1)dp[i]=max dp[j] +Sum[j] +2ij−j2 −i2, for (i−k≤j≤i−1)

咦？這裡是不是有點特別的感覺？這有沒有看起來像一個直線方程式？若把 iii 當作直線方程式的 xxx，有沒有像 y=ax+by = ax + by=ax+b ?

若寫的更清楚一點，在 y=ax+by = ax + by=ax+b 中:

a=2ia = 2ia=2i

b=dp[j]+Sum[j]−j2b = dp[j] + Sum[j] - j^2b=dp[j]+Sum[j]−j2

y=dp[i]+i2y = dp[i] + i^2y=dp[i]+i2 ，因此 y−i2y - i^2y−i2 就是我們這次的 dpdpdp 值。

那 deque 裡面的元素們就會是一個函數嘛，且注意到我們的 aaa 係數，他一定是遞增的喔。每加進去一條新的線，斜率又會比上一條還要更大，這是因為 iii 只會遞增，所以造成斜率也會是 單調遞增 的。這個時候，就滿足了使用 斜率優化 的條件。

為了以下為了說明的方便，我們可以拿以下三條線作比較:

$T_1: $ 倒數第二條線 (黑)

T2:T_2 :T2​: 最後一條線 (紅)

T3:T_3:T3​: 當前新的直線 (藍)

  • $NOTE: $ 以下的內容可以搭配下圖當作參考。

斜率優化跟單調對列其實還滿像的，若 在 deque 最前面的直線超過了給定的範圍 KKK ，就會老死。

但有些地方不太一樣喔，我們不能像單調對列 只有做 以下的事: 「對於新的直線的 yyy 值，若比 當前在 deque 裡面 最右邊的直線的 yyy 值還要大的話，那麼就往前砍掉比他還小的直線 」，這樣會出問題！如果新的線無法在當時取代最後一條線，那在 T3T_3T3​ 就失去了 在之後的 iii 能提早取代 T2T_2T2​ 的機會，意即，在往後的 iii 中，T3T_3T3​ 可能已經比 T1T_1T1​ 強了，但若是 T2T_2T2​ 還沒老死，那我們的 maxmaxmax 就會選到 T2T_2T2​ 而不是極值 T3T_3T3​ ，這個時候就會出 BUG。要處理這個問題的話，我們可以先轉換一下，有一個很好的思維，就是 「最後再來維護 deque 的最前面要是最大值」這個特性。只要在取用 maxmaxmax 值前，用一個 while 迴圈一直讓 dq[0] 跟 dq[1] 做 yyy 值比較， 若 dq[0] 小於 dq[1] 就 popfrontpop\ frontpop front ，到最後就會是我們當前要的直線了。

另一點比較不一樣的是，儘管 T1T_1T1​ 可能在當前無法直接取代 T2T_2T2​，然而，T3T_3T3​ 若跟 T1T_1T1​ 聯手，是有可能把前一條線給取代掉的喔。更詳細的說，再次強調 因為斜率一定是遞增的，看到交點就意味著前一個直線必定死亡，那麼，若 T3T_3T3​ 與 T2T_2T2​ 的 xxx 座標交點，小於 T2T_2T2​ 與 T1T_1T1​ 的 xxx 座標交點，這代表著，T2T_2T2​ 在把 T1T_1T1​ 給殺掉前，T3T_3T3​ 就已經把 T2T_2T2​ 給殺掉了。T2T_2T2​ 永遠不會有當最大值的時候， 因此可以直接把他 poppoppop 掉。此動作就是俗稱的 夾殺。

交點的 xxx 座標公式 可以藉由一點簡單的國中數學推導出來:

\displaylines{\large y = ax + b \\\ \large y' = cx + d \\\ \\\ \large ax + b = cx + d \\\ \large ax - cx = d - b \\\ \\\ \large x = \frac{d - b}{a - c}}

交點是求出來了，但是，這樣又會出一個問題，我們若把 T2T_2T2​ 給夾殺掉了，這個意思是 我們 能完全確定 T2T_2T2​ 用不到，也就是說，我們認為 「在 T1T_1T1​ 和 T3T_3T3​ 的交點以前，T1T_1T1​ 是最佳選擇，在之後的話則是 T3T_3T3​ 是最佳選擇」，但其實「在 T1T_1T1​ 和 T3T_3T3​ 的交點以前」有可能 T1T_1T1​ 已經老死了！這個時候我們就很不幸的把以後的最佳選擇給夾殺掉了。

那為了解決這個問題，我們必須多考慮 T1T_1T1​ 會提早死亡的可能。因此，對於一條直線的預設死期，就是就是該直線加進去的 xxx 座標 +K+\ K+ K ， 我們先讓他跟 T1T_1T1​ 與 T2T_2T2​ 的 xxx 座標交點取 minminmin ，再跟 T2T_2T2​ 與 T3T_3T3​ 的 xxx 座標交點比較，來決定是否該夾殺 T2T_2T2​ 。

** 對於每條線的死期，我們能再換另一個角度去想，若把線想成一個 線段，也許會更直觀一點 (?)

** 這題數字很大，每一個加跟乘都很容易爆 int ，因此一定要用 long long 。
